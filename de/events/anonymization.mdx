Ein sehr wichtiger Teil der datenschutzkonformen Einrichtung ist die **URL-Anonymisierung**. Da wir keine personenbezogenen Daten (PII) erfassen, müssen URLs, die persönliche Identifier enthalten (z.B. eine User-ID, ein Token oder eine Bestellnummer), maskiert oder entfernt werden.

Unser Script bietet dir dafür zwei mächtige Werkzeuge: Die Maskierung von URL-Pfaden (`data-anonymize-url`) und das Entfernen von Query-Parametern (`data-exclude-params`).

---

## 1. URL-Pfade anonymisieren

Wenn deine Website URLs wie `https://beispiel.de/profil/12345/bestellungen` nutzt, ist `12345` eine personenbezogene User-ID. Füge das Attribut `data-anonymize-url` zu deinem Script-Tag hinzu, um diese Daten lokal im Browser des Nutzers zu maskieren, bevor sie an uns gesendet werden.

### Option A: Der Auto-Detect Modus (Empfohlen)

Die einfachste Methode ist der automatische Modus. Wenn du den Wert auf `true` setzt, sucht das Script selbstständig nach gängigen ID-Formaten und ersetzt diese durch `/uuid`.

```html
<script
    defer
    src="https://analytics.bchic.de/script.js"
    data-website-id="DEINE_WEBSITE_ID"
    data-anonymize-url="true"
></script>
```

**Was wird automatisch erkannt?**
* **UUIDs:** `/orders/550e8400-e29b...` ➔ `/orders/uuid`
* **Hex-Hashes (24+ Zeichen):** `/commit/a1b2c3d4e5f6...` ➔ `/commit/uuid`
* **Numerische IDs (18+ Ziffern):** `/invoice/123456789012345678` ➔ `/invoice/uuid`
* **Alphanumerische Token (20+ Zeichen):** `/session/xK9mP2nQ8rT5...` ➔ `/session/uuid`

### Option B: Benutzerdefinierte Vorlagen (Templates)

Wenn deine IDs kürzer sind (z.B. Benutzernamen oder kurze IDs), kannst du benutzerdefinierte Vorlagen verwenden. Das Script nutzt eine Mustersprache, um dynamische Teile zu maskieren. Mehrere Regeln trennst du einfach durch ein Komma.

```html
<script
    defer
    src="https://analytics.bchic.de/script.js"
    data-website-id="DEINE_WEBSITE_ID"
    data-anonymize-url="/profil/{userId}/*, /order/{orderId}/*!">
</script>
```

#### Syntax für Vorlagen

| Symbol | Funktion | Beispiel Output |
| :--- | :--- | :--- |
| `{platzhalter}` | Ersetzt den dynamischen Teil durch den Platzhalternamen. | `/profil/{userId}` |
| *(kein Wildcard)* | **Exakter Treffer:** Die Länge der URL muss exakt mit der Vorlage übereinstimmen. | Eingabe: `/a/b/c` wird ignoriert, wenn Regel `/a/b` ist. |
| `/*` | **Rest beibehalten:** Behält alle Unterseiten nach dem Treffer bei. | Vorlage `/u/{id}/*` macht aus `/u/123/settings` ➔ `/u/id/settings` |
| `/*!` | **Rest abschneiden:** Entfernt alles, was nach dem Template kommt. | Vorlage `/u/{id}/*!` macht aus `/u/123/settings` ➔ `/u/id` |

## 2. Query-Parameter ausschließen

Oft verbergen sich personenbezogene Daten nicht im Pfad, sondern in der URL nach dem Fragezeichen (z.B. `?token=abc123` oder `?session=xyz`). Mit dem Attribut `data-exclude-params` kannst du diese Parameter gezielt entfernen oder leeren.

```html
<script
    defer
    src="https://analytics.bchic.de/script.js"
    data-website-id="DEINE_WEBSITE_ID"
    data-exclude-params="/auth/signup:token, /search:q="
></script>
```

### Syntax für Parameter-Regeln

Das Format ist immer `[Pfad-Vorlage]:[Parameter-Name][Aktion]`. Die Pfad-Vorlage unterstützt dieselben `{platzhalter}` und Wildcards (`/*`, `/*!`) wie oben beschrieben.

<CardGroup cols={2}>
    <Card title="Parameter komplett entfernen" icon="trash">
        **Format:** `/pfad:parameter`
        
        Der Parameter wird vollständig aus der URL gelöscht.
        * **Regel:** `/auth/verify:token`
        * **Original:** `/auth/verify?token=123&lang=de`
        * **Gesendet:** `/auth/verify?lang=de`
    </Card>
    <Card title="Wert leeren (Key behalten)" icon="eraser">
        **Format:** `/pfad:parameter=` (Beachte das Gleichheitszeichen am Ende)
        
        Der Key bleibt erhalten, aber der Wert wird geleert. Nützlich, wenn du tracken willst, *dass* gesucht wurde, aber nicht *was*.
        * **Regel:** `/search:q=`
        * **Original:** `/search?q=meine+suche&sort=asc`
        * **Gesendet:** `/search?q=&sort=asc`
    </Card>
</CardGroup>

## Überprüfung & Testing

Woher weißt du, ob alles korrekt funktioniert? Prüfe es direkt im Browser:

1. Öffne die **Entwicklertools** deines Browsers (F12).
2. Wechsle zum Tab **Netzwerk** (Network).
3. Lade eine Seite deiner Website, die eine zu anonymisierende URL oder Parameter enthält.
4. Suche nach der Anfrage an den Tracking-Endpoint (`analytics.bchic.de`).
5. Klicke auf den Request und schaue dir den **Payload** an.

<Check>
    Das Feld `url` im Payload sollte bereits die anonymisierte Form (z.B. `/profil/userId/...` oder entfernte Parameter) enthalten. Ist das der Fall, verlassen keine persönlichen Daten den Browser des Nutzers.
</Check>
